# 學生從 0 到 1 做產品的關鍵學習

從 0 到 1 的關鍵，不是先找到完美點子，而是快速降低不確定性。對我們團隊來說，最卡住的時刻通常是想在第一版上就解完所有未來問題。

## 改變進度的三個規則

### 1. 先定義明確完成標準

每個衝刺週期，我們只寫一句話定義成功。如果任務無法直接支持這句話，就先放回 backlog。

### 2. 在真實限制下設計

我們不等待理想工具或完整資料，而是根據現有條件來設計：

- 時間有限，
- 技能層級不一，
- 需求持續變動。

這反而逼出更好的產品判斷。

### 3. 每次 Demo 都用使用者價值檢查

內部展示時，我們固定問三個問題：

- 使用者結果是否比之前更清楚？
- 互動流程是否比手動方式更快？
- 第一次接觸的人能否在一分鐘內理解流程？

如果答案是否定，就優先調整範圍，而不是再加功能。

## 給新團隊的建議

- 最終產品決策要有明確負責人。
- 在開發完成前就安排回饋節點。
- 把第一版當成學習工具，而不是最終成品。

進步最快的團隊，通常是願意「先小步交付、再持續公開學習」的團隊。
