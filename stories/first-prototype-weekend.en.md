# Building Our First AI Prototype in One Weekend

A few weeks before the event, our team had many ideas and almost no alignment. We all wanted to use AI, but each person imagined a different product. Instead of debating features, we changed our goal: build the smallest version that proves one clear user value.

## What We Built

We focused on a student helper flow:

1. The user describes a study problem.
2. The app suggests a short action plan.
3. The user receives a simple follow-up checklist.

This gave us one complete loop to test, even though it was very small.

## How We Worked

We divided work into three tracks:

- Product track: define user scenario, output format, and success criteria.
- App track: build the UI flow and local state handling in SwiftUI.
- Prompt track: design and iterate AI prompts using real test inputs.

Every four hours, we merged progress and removed anything that did not improve the core user loop.

## Mistakes We Fixed Early

- We started with too many optional screens.
- Our first prompt outputs were too long and hard to act on.
- We delayed test sessions until late in the weekend.

After cutting scope and testing earlier, our output became much better.

## Key Takeaways

- Clarity beats complexity at the prototype stage.
- A small closed loop is better than a large unfinished concept.
- Fast feedback from real users prevents wasted build time.

Our next step is to improve reliability and add lightweight personalization without breaking the simple flow.
